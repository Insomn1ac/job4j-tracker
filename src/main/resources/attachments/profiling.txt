Произведено профилирование приложения job4j_tracker с помощью утилит jstat, jmap, jconsole.
Профилирование производилось на проекте с реализацией хранилища в памяти.
1). jmap : по результатам профилирования с помощью jmap видно лишь количество созданных объектов и занимаемый этими объектами объем памяти.
2). jstat : по результатам профилирования проекта с помощью jstat видно, как происходит сборка мусора в режиме реального времени.
    Наглядно показано, как при заполнении Eden-региона происходит minor GC (в jstat - YGC) и перемещение объектов в Survivor space.
    Также можно увидеть перемещение объектов между S0 и S1 регионами по мере срабатывания minor GC.
    При заполнении OldGen (в jstat - OU) происходит major GC (в jstat - FGC), также видно затраченное время на сборку мусора,
    как major GC (FGCT), так и minor GC (YGCT).
3). jconsole : удобный инструмент профилирования приложения, с визуализацией данных в виде графиков. Есть раздел "VM Summary", где видно
    текущий и максимальный размер памяти, выделенный под Heap, также указаны алгоритмы сборки мусора, количество их итераций и затраченное время (
    Name = 'Copy', Collections = 15, Total time spent = 5,654 seconds
    Name = 'MarkSweepCompact', Collections = 20, Total time spent = 1 minute
    )
